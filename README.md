# testforRMR

Я использовал Selenide + JUnit в своих тестах. Здесь меньше 10 тестов, но думаю такого количества достаточно чтобы определить мой уровень.

# Запуск

 `mvn clean test -Dos={os_name}`

 Параметр `-Dos={os_name}` определяет для какой системы будет запускаться chromedriver.

 Варианты:

 `osx`

 `linux`

# Структура тестов

В pages у меня находятся объекты страниц с описанием всех элеметов и методов, которые с ними взаимодействуют.
В support я обычно помещаю всякие дополнительные данные и надстройки над фреймворками.
Таким образом в tests у нас получаются компактные и легкочитаемые тесты.
